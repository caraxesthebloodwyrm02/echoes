---
  1. Core Architecture & Modules

  The project is a multimodal AI assistant platform with 8 interconnected intelligence modules (all in core_modules/):
  - Parallel Simulation Engine (concurrent scenario exploration)
  - Intent Awareness Engine (NLP with 28+ intent types)
  - Thought Tracking System (cognitive relationship mapping)
  - Personality Engine (adaptive emotional intelligence)
  - Humor Engine (context-aware humor)
  - Cross-Reference System (dynamic knowledge connection)
  - Catch & Release Caching (intelligent memory management)
  - Value System (ethical reasoning framework)

  Key Layers:
  - API: FastAPI server with WebSocket streaming (/ws/stream), REST endpoints, JWT + API key auth, and Prometheus metrics.
  - Agents: AI agent orchestration in app/agents/.
  - Resilience: Circuit breakers (pybreaker), rate limiting (SlowApi), and retry logic (tenacity).
  - Knowledge: Context and memory management in app/knowledge/.

  2. Current Workflow & Commands

  Run/Dev Commands:

  # Start API server
  python -m uvicorn api.main:app --host 0.0.0.0 --port 8000

  # Run interactive CLI
  python assistant_v2_core.py

  # Test suite
  python -m pytest tests/ -q --tb=short  # Full pytest
  python test_integration_quick.py      # Quick validation (~2s, no API calls)
  python test_integration.py            # Full integration (requires OpenAI key)

  # Code quality
  ruff check .          # Lint
  black .               # Format
  mypy .                # Type check

  Dependencies:

  - Python 3.12+
  - FastAPI, LangChain, OpenAI, Pydantic v2, structlog
  - pip install -e ".[dev]" or uv sync for development.

  ---
  3. Standards & Discipline

  Code Standards:

  - Type hints required everywhere.
  - Async-first for I/O operations.
  - Conventional commits (e.g., fix(api):, feat(assistant):).
  - Structured logging (no print() in production).
  - snake_case for files/functions.

  Safety Rules:

  - Consent-Based Licensing: All data processing must respect consent boundaries.
  - DCoC Provenance: AI-generated content must carry metadata (model, timestamp, confidence).
  - No eval()/exec()/pickle on untrusted input.
  - Quarantined modules require explicit approval before modification.

  Decision Logging:

  - Architectural decisions must be documented with rationale and alternatives.
